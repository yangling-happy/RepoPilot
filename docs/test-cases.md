# RepoPilot 功能测试用例

## 1. 文档生成模块 (Doc Pipeline)

### 1.1 增量刷新 (refresh)

| 用例ID | 功能点 | 前置条件 | 操作步骤 | 预期结果 | 优先级 |
|--------|--------|----------|----------|----------|--------|
| TC-DOC-001 | HEAD一致跳过 | 本地仓库已存在，HEAD与远程一致 | 调用 refresh 接口 | 返回 "No new commits"，不生成文档 | P0 |
| TC-DOC-002 | 正常增量生成 | 本地HEAD与远程不一致，有Java文件变更 | 调用 refresh 接口 | 生成变更文件的文档，返回新commit列表 | P0 |
| TC-DOC-003 | 无变更文件 | HEAD不一致，但无Java文件变更 | 调用 refresh 接口 | 创建任务但不生成文档文件 | P1 |
| TC-DOC-004 | 删除文件处理 | 远程删除了Java文件 | 调用 refresh 接口 | 标记文件为 "File deleted"，docFilePath为空 | P1 |
| TC-DOC-005 | 不支持的文件类型 | 变更文件为 .md/.txt 等 | 调用 refresh 接口 | 跳过该文件，不调用GitLab API读取内容 | P1 |
| TC-DOC-006 | 仓库不存在 | 本地仓库目录不存在 | 调用 refresh 接口 | 抛出异常，提示仓库不存在 | P0 |
| TC-DOC-007 | 多个commit增量 | 远程有多个新commit | 调用 refresh 接口 | 为所有变更文件生成文档 | P1 |

|--------|--------|----------|----------|----------|--------|
